# Use Cases

mirrord supports several distinct workflows for cloud-native development:

- **[Local Development with Cloud Context](local-development.md)** - Run your code locally while it communicates with real cloud services, receives real traffic, and uses real environment variables
- **[mirrord for CI](mirrord-for-ci.md)** - Run integration and end-to-end tests in CI against a real cloud environment
- **[Preview Environments](preview-environments.md)** - Deploy isolated, ephemeral pods in your cluster for team review and QA
- **[Using mirrord with AI](../using-mirrord-with-ai/README.md)** - Configure AI agents to use mirrord for cloud-context development
